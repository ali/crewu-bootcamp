A thing or two about Unix filesystems:

When accessing a computer through the command line you will be executing commands through a program called a shell. Shells are highly customizable and we will get more into that later. For now all that you need to know is that the prompt shows you where within the filesystem you are.

Filesystems are a tree of folders and files. At the top of every linux filesystem is a “root” folder referred to as “/”. When giving filepaths, if you are giving the “absolute path” from the top of your filesystem to the folder, then begin the file path with “/” the root folder. If you are using a “relative path” giving the route to the directory starting from your current directory, then do not begin the path with “/”

Absolute path example:

“/home/goodwin/homework/logic/hw1”

Relative path example: (I am in my home directory /home/goodwin)

“homework/logic/hw1”

To make your life easier the path to your home directory (/home/<username>) can be abbreviated as “~”.

When you log into a machine you will, by default be put in your home directory. (Unless you customize otherwise)
